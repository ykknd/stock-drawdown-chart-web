# Security Name Selector Requirements

## User Value

- ユーザーが銘柄コードを覚えていなくても、企業名・銘柄名から対象銘柄を選択できる。
- 複数銘柄を検索結果から選択でき、選択済み銘柄を確認しながら更新できる。
- J-Quants無料枠利用時の銘柄数をUIとAPIの両方で制限し、レート制限や利用枠消費を抑える。

## Requirements

- 銘柄指定UIは銘柄コード直接入力ではなく、企業名・銘柄名検索と複数選択を主導線にする。
- 候補一覧は `data/jp_security_names.csv` の `code,name` を参照する。
- 検索欄に入力した文字で、企業名・銘柄名を部分一致検索し、候補リストを絞り込む。
- 候補表示は `銘柄名（コード）` とし、類似名やETFを区別できるようにする。
- 選択済み銘柄はチップまたはリストで表示し、個別に削除できる。
- 更新ボタン押下時は、選択済み銘柄の `code` を既存の `symbols` として `/api/drawdowns` に送信する。
- 選択済み銘柄は `localStorage` にコード配列として保存し、再読み込み後に復元する。
- 既存の `drawdown-board-symbols` が残っている場合は、初回だけコード文字列を読み取り、CSVの銘柄一覧と照合して移行する。
- CSVに存在しないコードは選択済み銘柄として追加せず、noticeで知らせる。
- J-Quants無料枠ONかつJ-Quantsモードでは、選択数を最大5件に制限する。
- 無料枠ON時に5件選択済みの場合、追加候補は追加できない状態にし、最大5件までのnoticeを表示する。
- 既に6件以上選択済みの状態で無料枠ONに切り替えた場合、自動削除せず、更新ボタンをdisabledにして選択数を減らすよう促す。
- バックエンドでもJ-Quants無料枠ONかつJ-Quantsモードで5件を超える `symbols` をHTTP 400で拒否する。
- yfinanceモードでの日経平均 `^N225` 自動追加は内部処理として維持し、ユーザー選択リストには表示しない。

## Out of Scope

- J-Quants APIからの銘柄マスター同期。
- CSV編集UI。
- 銘柄のお気に入りグループ管理。
- 銘柄一覧のサーバーサイドページング。
- 銘柄コード手入力を主導線として残すこと。

## Acceptance Criteria

- `GET /api/securities` がCSVから `code,name` の一覧を返す。
- 検索欄に企業名・銘柄名の一部を入力すると候補が絞り込まれる。
- 候補クリックで選択済みリストに追加される。
- 選択済み銘柄を個別に削除できる。
- 更新ボタン押下時に選択済みコードだけが `symbols` として送信される。
- 選択済み銘柄が `localStorage` に保存され、再読み込み後に復元される。
- J-Quants無料枠ON時は5件を超えて追加できない。
- J-Quants無料枠ONで6件以上の `symbols` を送るとHTTP 400になる。
- J-Quants無料枠OFFまたはyfinanceモードでは既存API上限の20件まで選択できる。
