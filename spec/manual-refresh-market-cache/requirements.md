# Manual Refresh Market Cache Requirements

## User Value

- ユーザーが設定を調整するたびに外部API取得が走らないようにし、J-Quantsのレート制限と利用枠消費を抑える。
- 同じユーザー/APIキーscopeで同日に取得済みの銘柄データを再利用し、操作を軽くする。
- 公開Cloud Runとローカル開発でキャッシュ保存先を簡単に切り替えられる。

## Requirements

- `/api/drawdowns` はユーザーが「更新」ボタンを押した時だけ呼ばれる。
- 初回表示、期間変更、足種変更、テクニカル指標変更、J-Quants無料枠切替では自動取得しない。
- DD軸レンジとx軸ズームは取得不要の表示設定として即時反映する。
- データ取得に関係する設定変更後は、画面に「設定変更は未反映です。更新を押してください」相当のnoticeを表示する。
- キャッシュは日足OHLCの `PricePoint` 生データを対象にする。
- drawdown、回復力、ローソク足集約、テクニカル指標はキャッシュ後の生データから毎回再計算する。
- キャッシュはユーザー/APIキーscope単位で分離する。
- J-Quants APIキーの生値はキャッシュキー、ファイル名、ファイル内容、レスポンス、ログに保存しない。
- Cloud Run公開環境ではCloud Storageキャッシュを利用できる。
- ローカル環境ではCドライブ配下のローカルファイルキャッシュを利用できる。

## Out of Scope

- APIキーのDB保存。
- キャッシュ閲覧API。
- ユーザーが任意にキャッシュを削除するUI。
- 強制再取得ボタン。
- Cloud Storageバケットやライフサイクルルールの自動作成。

## Acceptance Criteria

- 設定変更だけでは `/api/drawdowns` が呼ばれない。
- 更新ボタン押下時のみ、現在の設定で `/api/drawdowns` が呼ばれる。
- 同日・同一scope・同一銘柄の取得はキャッシュを利用する。
- APIキーAとAPIキーBのキャッシュは共有されない。
- 2年取得済みなら、同日の6か月取得では外部APIを呼ばず範囲切り出しで応答する。
- Cloud Storage、local、memoryの保存先を環境変数で切り替えられる。
